# Вариант 04 — WBS (эпики → фичи → задачи)

- Эпик A. Модель данных и миграции
  - Фичи: схемы User/Habit/Goal/Entry/Reminder/Streak; миграции
  - Задачи: A1. Описать таблицы и связи; A2. Написать миграции; A3. Подготовить демо‑данные

- Эпик B. API и ядро приложения
  - Фичи: REST API (/auth, /habits, /goals, /entries, /reminders, /stats); валидация
  - Задачи: B1. Реализовать endpoints CRUD; B2. Реализовать аутентификацию (JWT); B3. Добавить пагинацию и фильтры

- Эпик C. Логика streaks и статистики
  - Фичи: алгоритм расчёта серий, календарь выполнения, сводные отчёты
  - Задачи: C1. Реализовать подсчёт streaks; C2. Тесты на корректность расчётов; C3. API для статистики

- Эпик D. UI / дашборд
  - Фичи: личный кабинет, календарь, графики прогресса, список привычек
  - Задачи: D1. Компонент календаря выполнения; D2. Страница привычки с деталями; D3. Дашборд с виджетами

- Эпик E. Напоминания и уведомления
  - Фичи: настройка напоминаний, отправка уведомлений (in‑app/push/email)
  - Задачи: E1. Реализовать модель Reminder; E2. Сервис отправки напоминаний; E3. Интеграция с WebSocket (бонус)

- Эпик F. Безопасность и роли
  - Фичи: аутентификация, авторизация, scope доступа (свои vs публичные данные)
  - Задачи: F1. Реализовать RBAC; F2. Тесты доступа; F3. Документация прав

- Эпик G. Окружение и CI/CD
  - Фичи: Docker, CI pipeline, тесты, деплой
  - Задачи: G1. Dockerfile; G2. Github Actions/CI; G3. Подготовить инструкции для демо
