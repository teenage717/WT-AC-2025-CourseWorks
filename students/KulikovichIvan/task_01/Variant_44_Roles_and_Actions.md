# Вариант 44 — Учебные квизы «Проверим быстро» — Роли и действия

## Краткое описание проекта

**MVP:** Платформа для создания и прохождения учебных квизов с быстрой обратной связью.  
Основные функции: создание квизов, вопросы с вариантами ответов, прохождение с таймером, автоматическая проверка, результаты с подсчетом баллов.

**Ключевые сущности:** Quiz, Question, Option, QuizAttempt, AttemptAnswer, User

**Приемка (MVP):** Пользователь может зарегистрироваться, выбрать квиз, пройти его за отведенное время, получить результат с баллами и правильными ответами.

---

## Роли и основные действия

### 1. Администратор (admin)

**Управление контентом:**

- Создание, редактирование и удаление квизов
- Добавление вопросов с вариантами ответов
- Настройка времени прохождения и балльной системы
- Активация и деактивация квизов

**Управление пользователями:**

- Просмотр списка пользователей
- Блокировка и разблокировка аккаунтов
- Назначение ролей

**Аналитика и отчетность:**

- Просмотр статистики по квизам
- Мониторинг активности пользователей
- Экспорт данных в различных форматах

**Системное администрирование:**

- Настройка параметров платформы
- Просмотр логов и ошибок
- Резервное копирование данных

---

### 2. Пользователь (user)

**Регистрация и профиль:**

- Регистрация на платформе
- Редактирование личных данных
- Смена пароля

**Прохождение квизов:**

- Просмотр доступных квизов
- Начало попытки прохождения
- Ответы на вопросы в ограниченное время
- Завершение попытки и просмотр результатов

**Личная аналитика:**

- Просмотр истории попыток
- Анализ прогресса по темам
- Сравнение своих результатов со средними

**Дополнительные функции:**

- Экспорт своих результатов
- Получение сертификатов об успешном прохождении
- Доступ к материалам для подготовки

---

### 3. Гость (guest)

**Ознакомительный доступ:**

- Просмотр списка доступных квизов (без деталей)
- Ознакомление с описанием платформы
- Регистрация на платформе

---

## Примеры сценариев использования

### Сценарий 1: Создание и публикация квиза (Администратор)

1. Администратор создает новый квиз «Основы Python»
2. Добавляет вопросы с вариантами ответов
3. Настраивает время прохождения и баллы
4. Активирует квиз для пользователей

### Сценарий 2: Прохождение квиза (Пользователь)

1. Пользователь входит в систему
2. Выбирает квиз
3. Начинает попытку, запускается таймер
4. Отвечает на вопросы
5. Завершает попытку
6. Получает результат и анализ ошибок

### Сценарий 3: Анализ эффективности (Администратор)

1. Администратор просматривает статистику по квизу
2. Анализирует сложные вопросы
3. Вносит изменения в контент
4. Экспортирует данные для отчетов

### Сценарий 4: Отслеживание прогресса (Пользователь)

1. Пользователь заходит в профиль
2. Анализирует график прогресса
3. Экспортирует достижения
4. Выбирает следующий квиз
