# Вариант 44 — Учебные квизы «Проверим быстро» — Маршруты (user flows)

## 1. Идентификация и регистрация

- **Гость → Регистрация → Пользователь**
  - Посещение главной страницы
  - Нажатие "Зарегистрироваться"
  - Заполнение формы: email, имя пользователя, пароль
  - Подтверждение регистрации по email (опционально)
  - Автоматический вход в систему

- **Гость/Пользователь → Вход → Сессия**
  - Ввод email и пароля
  - Получение JWT токена
  - Перенаправление на дашборд

## 2. Обзор и выбор квиза

- **Пользователь → Просмотр квизов → Выбор**
  - Отображение списка доступных квизов
  - Фильтрация по категориям, сложности, времени
  - Просмотр деталей квиза: описание, время, количество вопросов
  - Начало прохождения квиза

## 3. Прохождение квиза

- **Пользователь → Начало попытки → Прохождение**
  - Подтверждение начала попытки
  - Запуск таймера обратного отсчета
  - Постраничная навигация по вопросам
  - Выбор вариантов ответов
  - Возможность пропустить и вернуться позже
  - Просмотр оставшегося времени

- **Пользователь → Завершение попытки → Результаты**
  - Досрочное завершение или по истечении времени
  - Автоматическая проверка ответов
  - Расчет баллов и процента правильных ответов
  - Отображение правильных ответов и пояснений
  - Возможность просмотреть детали по каждому вопросу

## 4. Анализ результатов

- **Пользователь → Личная статистика → Анализ**
  - Просмотр истории всех попыток
  - Графики прогресса по времени
  - Сравнение результатов с предыдущими попытками
  - Рекомендации для улучшения
  - Экспорт результатов в различных форматах

## 5. Администрирование (Администратор)

- **Администратор → Создание квиза → Публикация**
  - Создание нового квиза с базовой информацией
  - Добавление вопросов с вариантами ответов
  - Настройка параметров: время, баллы, проходной балл
  - Тестовое прохождение квиза
  - Активация квиза для пользователей

- **Администратор → Управление пользователями → Модерация**
  - Просмотр списка пользователей
  - Блокировка/разблокировка аккаунтов
  - Сброс паролей
  - Назначение административных прав

- **Администратор → Аналитика платформы → Отчеты**
  - Просмотр общей статистики
  - Анализ популярности квизов
  - Выявление проблемных вопросов
  - Генерация отчетов для руководства

## 6. Социальные функции (бонус)

- **Пользователь → Достижения → Публикация**
  - Получение достижений за успехи
  - Публикация результатов в социальных сетях
  - Сравнение с друзьями (если реализована система друзей)
  - Участие в рейтингах

## 7. Уведомления и напоминания

- **Система → Уведомления → Пользователь**
  - О новых квизах по интересующим темам
  - О необходимости продолжить начатый квиз
  - О достижениях и рекордах
  - О технических обновлениях платформы

## Поток данных между компонентами

Frontend (Vue.js) ←→ API Gateway (FastAPI) ←→ Бизнес-логика ←→ База данных (PostgreSQL)
↑ ↑ ↑
└── WebSocket (реальное время) └── Аутентификация (JWT) └── Кэширование (Redis)

Mobile App → REST API → Бизнес-логика → База данных
↑ ↑
└── Push-уведомления

## Критические пути пользователя

1. **Быстрое начало:** Гость → Регистрация → Выбор квиза → Начало → 3 минуты
2. **Полное прохождение:** Вход → Выбор → Прохождение → Результаты → Анализ → 15-30 минут
3. **Административная задача:** Вход → Создание квиза → Добавление вопросов → Тестирование → Публикация → 30-60 минут
